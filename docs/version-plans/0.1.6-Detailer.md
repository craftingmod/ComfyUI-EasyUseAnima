# EasyUseAnima 0.1.6 Anima Detailer 구현 계획

## 1. 목표

`Anima Detailer`는 ComfyUI Impact Pack의 `DetailerForEach` 동작을 최대한 동일하게 재현하되, ANIMA 워크플로우에서 자주 쓰는 최적화와 특수 조건을 안전하게 다루기 위한 디테일러 노드다.

핵심 목표는 다음이다.

```text
1. Impact Pack DetailerForEach의 입력 구조와 동작 원리를 최대한 유지한다.
2. SEGS 기반 crop → sampling → paste 흐름을 동일하게 재현한다.
3. sampling backend도 Impact Pack과 같은 방식으로 재현한다.
4. Spectrum 가속, Anima Mod Guidance, tiled encode/decode, scheduler_func_opt가 끊기지 않게 한다.
5. 16채널 VAE / 특수 VAE에서 crop size mismatch가 나지 않도록 안전장치를 추가한다.
6. Impact Pack이 없어도 EasyUseAnima 전체 import는 실패하지 않게 한다.
```

이번 구현의 핵심은 “Impact Pack 대체”가 아니라 “ANIMA 워크플로우용 Impact-compatible Detailer”다.

---

## 2. 구현 방향 정정

이전 설계에서는 sampling backend를 native로 분리하는 방안도 고려했다. 하지만 현재 목표에서는 native sampling 재구현보다 다음 방향이 더 안전하다.

```text
Impact Pack DetailerForEach의 detail loop를 재현한다.
Impact Pack core.enhance_detail()의 sampling 흐름도 재현한다.
Impact Pack impact_sampling.ksampler_wrapper() 방식도 가능한 한 동일하게 재현한다.
```

즉, `Anima Detailer`는 다음 구조를 가진다.

```text
Anima Detailer
  ├─ Impact-compatible input schema
  ├─ SEGS scale match / segment loop
  ├─ crop image 생성
  ├─ mask feather / noise mask 처리
  ├─ conditioning mask crop
  ├─ enhance_detail equivalent
  │   ├─ wildcard / LoRA 처리
  │   ├─ guide_size / max_size resize
  │   ├─ VAE encode 또는 InpaintModelConditioning
  │   ├─ Impact-style ksampler_wrapper
  │   ├─ VAE decode 또는 tiled decode
  │   └─ 원래 crop 크기로 resize
  └─ pixel-space paste
```

## 5. 전체 처리 흐름

### 5.1 실행 시작

```text
1. image batch 확인
2. image clone
3. SEGS와 image 크기 match
4. wildcard ordering 처리
5. differential diffusion 필요 여부 확인
6. segment loop 시작
```

Impact Pack과 동일하게 image batch는 우선 허용하지 않는다.

```python
if len(image) > 1:
    raise Exception("Anima Detailer does not allow image batches.")
```

## 6. SEGS 처리

### 6.1 SEGS scale match

입력 image와 `SEGS` 기준 크기가 다를 수 있으므로 먼저 scale match를 수행한다.

```python
segs = segs_scale_match(segs, image.shape)
```

초기 구현에서는 Impact Pack helper를 직접 사용할 수 있으면 동적 import해서 사용한다.

```python
def impact_segs_scale_match(segs, image_shape):
    core = load_impact_core()
    return core.segs_scale_match(segs, image_shape)
```

나중에 Impact 의존을 줄이려면 내부 구현으로 대체한다.

### 6.2 segment 순서

wildcard 정렬 모드를 지원한다.

```text
ASC
  bbox x, y 기준 오름차순

DSC
  bbox x, y 기준 내림차순

ASC-SIZE
  bbox 면적 오름차순

DSC-SIZE
  bbox 면적 내림차순

기본값
  SEGS 원래 순서
```

### 6.3 crop image 생성

중요한 규칙:

```text
seg.cropped_image를 그대로 쓰지 않는다.
현재까지 paste된 image에서 seg.crop_region을 다시 crop한다.
```

이렇게 해야 겹치는 segment가 있을 때 앞 segment의 수정 결과가 뒤 segment에 반영된다.

## 7. mask 처리

### 7.1 paste mask

`seg.cropped_mask`를 tensor로 변환한 뒤 `feather`만큼 blur 처리한다.

```text
paste_mask = gaussian_blur(seg.cropped_mask, feather)
```

이 mask는 sampling noise mask가 아니라 최종 paste용 mask다.

### 7.2 noise mask

```text
noise_mask = true
  seg.cropped_mask를 latent noise_mask로 사용

noise_mask = false
  crop 전체 img2img에 가까운 방식으로 처리
```

```python
cropped_mask = seg.cropped_mask if noise_mask else None
```

### 7.3 noise_mask_feather

`noise_mask_feather > 0`이면 latent noise mask에 별도 feather를 적용한다.

```text
feather
  최종 paste 경계용

noise_mask_feather
  latent sampling mask 경계용
```

둘은 역할이 다르므로 분리한다.

## 8. conditioning 처리

### 8.1 positive / negative crop

conditioning 내부에 `mask` key가 있으면 원본 image 기준 mask를 segment crop region 기준으로 잘라낸다.

```python
cropped_positive = crop_conditioning_masks(positive, image, seg.crop_region)
cropped_negative = crop_conditioning_masks(negative, image, seg.crop_region)
```

규칙:

```text
conditioning item의 details 안에서 key == "mask"인 값만 crop한다.
나머지 metadata는 유지한다.
```

### 8.2 metadata 보존

ANIMA 쪽에서는 conditioning 안에 model patch, guidance 관련 metadata, regional prompt 정보가 들어갈 수 있으므로 다음을 보장한다.

```text
- mask 외 metadata를 삭제하지 않는다.
- 알 수 없는 key를 임의로 필터링하지 않는다.
- conditioning list 구조를 유지한다.
```

### 8.3 Mod Guidance 대응

초기 구현에서는 별도 Mod Guidance branch를 강제로 만들지 않는다.

우선 정책:

```text
positive / negative / model 안에 이미 적용된 Mod Guidance 정보를 보존한다.
conditioning metadata를 복사할 때 손실시키지 않는다.
wildcard 처리 중 positive를 완전히 덮어쓰는 경우를 주의한다.
```

추후 필요하면 `anima_mod_guidance_opt` 입력을 추가한다.

## 9. wildcard 처리

Impact Pack과 같은 동작을 목표로 한다.

### 지원 문법

```text
[CONCAT]
  wildcard positive를 기존 positive에 concat한다.

[SKIP]
  현재 segment를 건너뛴다.

[STOP]
  현재 segment 이후 처리를 중단한다.

ASC / DSC / ASC-SIZE / DSC-SIZE
  segment 처리 순서 변경
```

### 처리 순서

```text
1. wildcard 전체 문자열 파싱
2. segment별 wildcard item 선택
3. segment seed 결정
4. [SKIP]이면 continue
5. [STOP]이면 break
6. enhance_detail 단계로 wildcard item 전달
```

### 주의사항

wildcard 처리 중 `positive`가 새 conditioning으로 교체될 수 있다. 이때 ANIMA 관련 conditioning metadata가 사라지지 않도록 `preserve_conditioning_metadata` 옵션을 둔다.

초기 구현에서는 Impact Pack 방식과 동일하게 동작시키고, metadata 손실이 확인되면 보존 로직을 추가한다.

## 11. crop resize 정책

### 11.1 guide_size 기준

```text
guide_size_for = bbox
  bbox의 짧은 변이 guide_size가 되도록 확대 배율 계산

guide_size_for = crop_region
  crop image의 짧은 변이 guide_size가 되도록 확대 배율 계산
```

### 11.2 max_size 제한

확대 후 width 또는 height가 `max_size`를 넘으면 다시 축소한다.

```text
긴 변 <= max_size
```

### 11.3 force_inpaint

```text
force_inpaint = false
  이미 충분히 큰 영역이면 skip 가능

force_inpaint = true
  확대가 필요 없어도 crop sampling 수행
```

### 11.4 alignment

ANIMA에서는 16채널 VAE 또는 특수 VAE 대응을 위해 최종 sampling 크기를 alignment에 맞춘다.

```text
alignment = none
  Impact Pack 원본과 동일하게 int size 사용

alignment = 8
  일반 SD 계열 안전값

alignment = 16
  일부 특수 VAE 후보

alignment = 32
  ANIMA 기본 권장값

alignment = 64
  더 보수적인 크기 정렬
```

초기 기본값:

```text
alignment = 32
```

helper:

```python
def align_size(width, height, alignment):
    if alignment in (None, "none", 0):
        return width, height

    align = int(alignment)
    width = max(align, round(width / align) * align)
    height = max(align, round(height / align) * align)
    return width, height
```

주의:

```text
Impact Pack 완전 재현 모드가 필요하면 alignment = none을 사용한다.
ANIMA 기본 모드에서는 alignment = 32를 사용한다.
```

## 13. sampling backend

이번 구현에서는 sampling backend도 Impact Pack 방식을 최대한 동일하게 재현한다.

### 13.1 함수 구조

```python
def anima_ksampler_wrapper(
    model,
    seed,
    steps,
    cfg,
    sampler_name,
    scheduler,
    positive,
    negative,
    latent_image,
    denoise,
    refiner_ratio=None,
    refiner_model=None,
    refiner_clip=None,
    refiner_positive=None,
    refiner_negative=None,
    noise=None,
    scheduler_func=None,
    sampler_opt=None,
):
    ...
```

초기에는 refiner 계열은 지원하지 않거나, Impact Pack 구조를 참고해 나중에 추가한다.

### 13.2 denoise 처리

Impact Pack 방식 재현:

```text
advanced_steps = floor(steps / denoise)
start_at_step = advanced_steps - steps
end_at_step = start_at_step + steps
```

예:

```text
steps = 20
denoise = 0.5

advanced_steps = 40
start_at_step = 20
end_at_step = 40
```

즉 전체 40 step schedule을 만들고 후반 20 step 구간만 샘플링한다.

### 13.3 sigma 계산

우선순위:

```text
scheduler_func가 있음
  scheduler_func(model, sampler_name, steps) 사용

scheduler_func가 없음
  Impact Pack과 같은 calculate_sigmas 방식 사용
```

추가 scheduler가 필요한 경우 Impact Pack의 scheduler 목록을 참고한다.

### 13.4 sampler 생성

Impact Pack과 같은 정책을 따른다.

```text
sampler_opt 있음
  sampler_opt 사용

sampler_opt 없음
  sampler_name으로 ComfyUI sampler object 생성
```

SDE 계열 sampler는 Brownian noise sampler wrapper가 필요할 수 있으므로 Impact Pack 방식을 참고한다.

### 13.5 custom noise

`detailer_hook.get_custom_noise()`가 있으면 해당 noise를 사용한다.

없으면 seed 기반 noise를 생성한다.

### 13.6 sample 실행

기본적으로 ComfyUI의 `sample_custom` 경로를 사용한다.

```python
comfy.sample.sample_custom(
    model,
    noise,
    cfg,
    sampler,
    sigmas,
    positive,
    negative,
    latent_image,
    noise_mask=noise_mask,
    callback=callback,
    disable_pbar=disable_pbar,
    seed=seed,
)
```

### 13.7 scheduler_func_opt 보존

`scheduler_func_opt` 입력은 sampling backend까지 그대로 전달한다.

```text
Detailer input scheduler_func_opt
  → anima_enhance_detail(... scheduler_func=...)
  → anima_ksampler_wrapper(... scheduler_func=...)
  → sigma 계산에 사용
```

이 경로가 끊기면 Spectrum / custom scheduler 계열 최적화가 동작하지 않을 수 있다.

`cycle` 값만큼 같은 crop latent를 반복 sampling한다.

```text

2. detailer_hook.cycle_latent()
3. detailer_hook.pre_ksample()
```

hook이 없으면 기본값 그대로 진행한다.

cycle별 seed:

```text

```

## 16. crop 복원과 paste

sampling 결과 image는 다시 원래 crop 크기로 resize한다.

```text

  → original crop width/height로 resize
  → paste mask를 사용해 원본 image에 tensor_paste
```

```text

```

이 방식이 Impact Pack의 품질 정책과 맞다.

## 18. Spectrum / Mod Guidance 대응 정책

### 18.1 기본 판단

sampling backend를 Impact Pack과 같은 방식으로 재현하면, Spectrum / Mod Guidance 문제는 대부분 다음 두 지점에서 발생할 가능성이 높다.

```text
1. model patch 또는 model_options가 crop sampling까지 보존되지 않음
2. scheduler_func_opt / sampler_opt / hook 경로가 detailer 내부에서 끊김
```

따라서 Anima Detailer의 우선 목표는 새 최적화 API를 만드는 것이 아니라 다음을 보장하는 것이다.

```text
- 입력 model을 clone/교체할 때 model patch를 잃지 않는다.
- model.model_options를 임의로 초기화하지 않는다.
- scheduler_func_opt를 sampling backend까지 전달한다.
- detailer_hook의 custom sampler / pre_ksample 결과를 반영한다.
- conditioning metadata를 crop 과정에서 삭제하지 않는다.
```

### 18.2 Mod Guidance

초기 구현 정책:

```text
Mod Guidance가 model patch 방식이면 model을 그대로 보존한다.
Mod Guidance가 conditioning metadata 방식이면 positive metadata를 보존한다.
Mod Guidance가 sampler hook 방식이면 detailer_hook 또는 scheduler_func_opt로 전달되도록 한다.
```

별도 `mod_guidance_quality` 문자열을 detailer 내부에서 다시 encode하는 방식은 1차 구현에서 제외한다.
먼저 기존 workflow에서 이미 생성된 conditioning/model patch를 보존하는 것에 집중한다.

### 18.3 Spectrum

초기 구현 정책:

```text
Spectrum이 model patch 방식이면 model을 그대로 보존한다.
Spectrum이 scheduler_func_opt 방식이면 scheduler_func_opt를 sampling backend까지 전달한다.
Spectrum이 custom sampler 방식이면 detailer_hook.get_custom_sampler() 또는 sampler_opt 경로를 유지한다.
```

아직 타입이 확정되지 않은 `spectrum_opt` 입력은 1차 구현에서 만들지 않는다.
필요성이 확인되면 0.1.7에서 추가한다.

## 19. Impact Pack 의존성 정책

### 19.1 import 정책

EasyUseAnima import 시점에 Impact Pack을 import하지 않는다.

나쁜 예:

```python
from impact import core
from impact import impact_sampling
```

좋은 예:

```python
def load_impact_core():
    try:
        import impact.core as core
        return core
    except Exception as exc:
        raise RuntimeError(
            "Anima Detailer requires ComfyUI Impact Pack for this backend."
        ) from exc
```

### 19.2 1차 구현 선택지

두 가지 방식 중 하나를 선택한다.

#### Option A. Impact backend 직접 호출

```text
장점:
  빠르게 동작
  Impact Pack과 결과 동일

단점:
  내부 sampling 수정 어려움
  Spectrum/Mod Guidance 문제를 그대로 가질 수 있음
```

#### Option B. Impact code를 참고해 내부 재현

```text
장점:
  sampling backend를 제어 가능
  ANIMA 안전장치 추가 가능
  Spectrum/Mod Guidance 경로 보존 가능

단점:
  구현량 많음
  Impact Pack 업데이트와 차이 발생 가능
```

이번 목표는 Option B다.

단, 개발 중 비교용으로 Option A backend를 남겨도 된다.

```text
backend = impact
backend = anima_reproduce
```

기본값:

```text
backend = anima_reproduce
```

## 22. 검증 계획

### 기본 동작

```text
- SEGS 1개 얼굴 detail
- SEGS 여러 개 얼굴 detail
- empty mask segment skip
- overlapping segment 처리
- noise_mask on/off
- force_inpaint on/off
- feather 값 변화
```

### Impact 비교

```text
- 같은 image, 같은 SEGS, 같은 seed, 같은 sampler 설정으로 Impact DetailerForEach와 비교
- backend=impact와 backend=anima_reproduce 결과 비교
- 큰 차이가 나면 resize, sigma, noise, paste 단계부터 비교
```

### sampling 검증

```text
- denoise=1.0
- denoise=0.5
- denoise=0.3
- scheduler_func_opt 연결
- detailer_hook pre_ksample 동작
- tiled_encode on/off
- tiled_decode on/off
```

### ANIMA 최적화 검증

```text
- Spectrum model patch가 적용된 model 입력
- Anima Mod Guidance model patch가 적용된 model 입력
- conditioning metadata가 있는 positive 입력
- regional conditioning mask가 있는 positive 입력
```

### 16채널 VAE 검증

```text
- alignment=none
- alignment=8
- alignment=32
- tiled_encode=true
- tiled_decode=true
- 작은 crop
- 홀수 크기 crop
- 32 배수가 아닌 crop
```

## 25. EasyUse Anima Wildcard 구현 체크리스트

Anima Detailer의 segment별 wildcard와 별개로, EasyUseAnima 노드팩 공통에서 사용할 수 있는 와일드카드 기능을 추가한다.

목표:

```text
1. 프롬프트 입력에서 와일드카드 문법을 안정적으로 확장한다.
2. 워크플로우 저장/공유 시 재현 가능한 결과를 유지한다.
3. Prompt Studio / Prompt Studio Advanced / Prompt Builder와 충돌하지 않게 한다.
4. 기존 자동완성, 하이라이트, 프롬프트 교정 흐름과 일관되게 동작한다.
```

### 25.1 저장 위치

```text
__easyuse_anima__/wildcards/
  *.txt
  하위 폴더 허용
```

실제 사용자 데이터 저장은 기존 저장 정책을 따른다.

```text
folder_paths.get_system_user_directory("easyuse_anima")/wildcards
```

ComfyUI user data API를 사용할 수 없는 환경에서는 노드팩 내부 `__easyuse_anima__/wildcards`를 fallback으로 사용한다.

### 25.2 문법 초안

```text
__name__
  wildcards/name.txt에서 1개 항목 선택

__folder/name__
  wildcards/folder/name.txt에서 1개 항목 선택

{a|b|c}
  inline 선택지 중 1개 선택
```

초기 구현에서는 nested wildcard를 제한적으로 지원한다.

```text
max_depth = 8
```

순환 참조가 감지되면 명확한 오류를 낸다.

### 25.3 재현성 정책

와일드카드 결과는 seed 기반으로 결정한다.

```text
입력 prompt
wildcard 파일 내용
seed
wildcard mode
```

위 값이 같으면 같은 결과가 나와야 한다.

이미지 워크플로우로 저장된 경우에는 확장된 최종 프롬프트를 저장해 재현 가능하게 한다.

### 25.4 노드 계획

초기 후보:

```text
Anima Wildcard Prompt
  input:
    prompt: STRING
    seed: INT
    freeze_output: BOOLEAN
    cached_prompt: STRING

  output:
    prompt: STRING
    report: STRING
```

추후 후보:

```text
Anima Wildcard Manager
  wildcard 파일 목록 확인
  누락 wildcard report
  선택지 preview
```

### 25.5 Prompt Studio 연동

```text
- Prompt Studio Advanced에서 와일드카드 확장 전 텍스트와 확장 후 텍스트를 구분한다.
- 하이라이트와 오타 표시는 기본적으로 확장 후 텍스트 기준으로 적용한다.
- 필요하면 raw prompt / expanded prompt를 둘 다 report에 남긴다.
- 자동완성은 wildcard 토큰 내부에서는 과하게 개입하지 않는다.
```

## 26. Codex 작업 지시문

```text
EasyUseAnima 0.1.6 Anima Detailer 구현.

목표:
Impact Pack DetailerForEach의 detail loop와 sampling backend를 최대한 동일하게 재현한 Anima Detailer를 만든다.
기존 Impact Pack Detailer에서 Spectrum 가속, Anima Mod Guidance, 특수 VAE/16채널 VAE 환경에서 문제가 생기는 것을 줄이기 위해 model/conditioning/scheduler/hook 경로를 보존한다.

1. EasyUseAnimaDetailer 노드를 추가한다.
   CATEGORY = "EasyUse Anima/Detailer"
   RETURN_TYPES = ("IMAGE",)
   FUNCTION = "doit"

2. 입력은 Impact Pack DetailerForEach와 최대한 동일하게 맞춘다.
   required:
   image, segs, model, clip, vae,
   guide_size, guide_size_for, max_size,
   seed, steps, cfg, sampler_name, scheduler,
   positive, negative,
   denoise, feather, noise_mask, force_inpaint,
   wildcard, cycle

   optional:
   detailer_hook,
   inpaint_model,
   noise_mask_feather,
   scheduler_func_opt,
   tiled_encode,
   tiled_decode

   Anima 추가:
   alignment = ["none", "8", "16", "32", "64"], default "32"
   preserve_conditioning_metadata = true
   fail_on_unsupported_opt = true

3. EasyUseAnima import 시점에 Impact Pack을 top-level import하지 않는다.
   필요한 Impact helper는 실행 시점에 동적 import한다.
   Impact Pack 미설치 상태에서도 기존 EasyUseAnima 노드는 import되어야 한다.

4. DetailerForEach.do_detail 흐름을 내부 구현으로 재현한다.
   - image batch 제한
   - image clone
   - segs_scale_match
   - wildcard ordering
   - segment loop
   - 현재 image에서 crop_region 기준 crop 생성
   - seg.cropped_mask feather
   - noise_mask 처리
   - positive/negative conditioning mask crop
   - [SKIP]/[STOP] 처리
   - enhance_detail equivalent 호출
   - pixel-space tensor_paste
   - detailer_hook.post_paste 지원

5. core.enhance_detail 흐름을 내부 구현으로 재현한다.
   - wildcard / LoRA 처리
   - guide_size / max_size 기준 resize 크기 계산
   - force_inpaint skip 조건
   - alignment 적용
   - upscaled image 생성
   - control_net_wrapper 적용
   - VAE encode 또는 InpaintModelConditioning
   - noise_mask latent 추가
   - detailer_hook post_encode/pre_ksample/pre_decode/post_decode 지원
   - cycle loop
   - sampling backend 호출
   - VAE decode 또는 tiled decode
   - 원래 crop 크기로 resize

6. sampling backend는 Impact Pack ksampler_wrapper 방식을 재현한다.
   - denoise 기반 advanced_steps 계산
   - start_at_step / end_at_step 계산
   - scheduler_func_opt가 있으면 sigma 계산에 사용
   - sampler_opt 또는 detailer_hook custom sampler를 지원
   - custom noise 지원
   - comfy.sample.sample_custom 경로 사용
   - noise_mask 전달
   - positive/negative conditioning metadata를 보존

7. Spectrum / Mod Guidance 대응은 새 API를 만들기보다 기존 객체 보존을 우선한다.
   - model patch를 잃지 않는다.
   - model.model_options를 임의 초기화하지 않는다.
   - positive/negative conditioning metadata를 삭제하지 않는다.
   - scheduler_func_opt를 sampling backend까지 전달한다.
   - detailer_hook custom sampler/pre_ksample 결과를 반영한다.

8. 16채널 VAE 대응을 위해 alignment helper를 추가한다.
   - 기본 alignment는 32
   - Impact 완전 재현이 필요하면 alignment="none" 사용
   - size mismatch 발생 시 명확한 오류와 로그를 남긴다.

9. 0.1.6에서 제외한다.
   - MaskToSEGS 대체 노드
   - detector 내장
   - batch image 지원
   - 별도 spectrum_opt 타입
   - detailer 내부 Mod Guidance 문자열 encode
   - refiner 완전 지원

10. 검증한다.
   - Impact DetailerForEach와 같은 입력 연결 가능
   - 일반 SEGS detail 동작
   - denoise 0.3/0.5/1.0 동작
   - scheduler_func_opt 전달 확인
   - tiled_encode/tiled_decode 동작
   - Spectrum model patch 입력 시 model 보존
   - Mod Guidance 적용 model/conditioning 입력 시 metadata 보존
   - 16채널 VAE에서 alignment=32로 size mismatch 감소
   - Impact Pack 미설치 상태에서 기존 EasyUseAnima import 성공
```
